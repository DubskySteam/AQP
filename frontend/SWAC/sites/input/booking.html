Examples

Example 1: Show available rooms

Booking component allows to list items, such as rooms, devices or other stuff available for booking. Useing a second datasource with made bookings it's possible to show the current booking state.
The bookingdata entered by the user can be send to a seperate script, allowing it to use enhanced backend scripts, like sending a verification email. 


<div id="booking_example1" swa="Booking FROM ../data/booking/rooms.json"></div>

Example 2: One item that can be booked from more than one person

If the bookable items contain the "places" attribute the item can be booked by more than one person at one time.
Booking components expected attributes are compatible with those of the Event component, makeing it possible to use one datasource for listing and booking events. 


<div id="booking_example1" swa="Booking FROM ../data/events/exampleevents.json"></div>
